# <a name="troubleshoot-quarantine-state"></a>Řešení problémů se stavem karantény

Služba zřizování služby Azure Active Directory (AD) monitoruje stav konfigurace. Umístí také poškozené aplikace do **karanténního** stavu. Pokud máte nejvíc nebo všechny hovory prováděné vůči cílovému systému konzistentně, pak je úloha zřizování označená jako **v karanténě**. Příkladem chyby je **přijatá chyba z důvodu neplatných přihlašovacích údajů správce**. Další informace najdete v tématu [zřizování aplikací ve stavu karantény](https://docs.microsoft.com/azure/active-directory/app-provisioning/application-provisioning-quarantine-status).

Informace o řešení potíží s synchronizací cloudu najdete v tématu [zajišťování problémů v karanténě](https://docs.microsoft.com/azure/active-directory/cloud-sync/how-to-troubleshoot#provisioning-quarantined-problems).
